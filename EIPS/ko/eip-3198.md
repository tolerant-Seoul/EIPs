## 간단한 요약
EVM이 블록의 기본 수수료에 접근할 수 있게 하는 옵코드를 추가합니다.

## 요약

실행 중인 현재 블록의 기본 수수료 값을 반환하는 `BASEFEE (0x48)` 옵코드를 추가합니다.

## 동기
의도된 사용 사례는 컨트랙트가 기본 수수료 값을 얻을 수 있도록 하는 것입니다. 이 기능은 다음과 같은 기존 사용 사례를 가능하게 하거나 개선합니다:
- 누구든 트랜잭션으로 "찌르도록" 현상금을 설정해야 하는 컨트랙트는 현상금을 `BASEFEE + x` 또는 `BASEFEE * (1 + x)`로 설정할 수 있습니다. 이렇게 하면 시장 상황에 관계없이 항상 "충분히" 지불하므로 메커니즘이 더 신뢰할 수 있게 됩니다.
- 이를 기반으로 가스 선물을 구현할 수 있습니다. 이것은 가스토큰보다 더 정확합니다.
- 상태 채널, 플라즈마, 옵티롤 및 기타 사기 증명 기반 솔루션의 보안을 개선합니다. `BASEFEE`를 입력으로 가지면 `BASEFEE`가 높은 것을 볼 때 챌린지 기간을 자동으로 연장할 수 있습니다.

## 사양
`G_base` 가스 비용으로 `(0x48)`에 `BASEFEE` 옵코드를 추가합니다.

|  Op  	| 입력 	| 출력 	| 비용 	|
|:----:	|:-----:	|:------:	|:----:	|
| 0x48 	|   0   	|    1   	|   2  	|

## 근거

### 가스 비용
기본 수수료 값은 트랜잭션을 처리하는 데 필요합니다. 이는 EVM 코드를 실행하기 전에 해당 값이 이미 사용 가능함을 의미합니다.
옵코드는 추가적인 복잡성과 추가 읽기/쓰기 작업을 추가하지 않으므로 `G_base` 가스 비용을 선택합니다.

## 하위 호환성
이 옵코드와 관련하여 알려진 하위 호환성 문제는 없습니다.

## 테스트 케이스

### 일반적인 경우
현재 블록 기본 수수료가 `7 wei`라고 가정합니다.
이것은 값 `7`(왼쪽 패딩된 byte32)을 스택에 푸시해야 합니다.

바이트코드: `0x4800` (`BASEFEE, STOP`)

|  Pc   |      Op     | 비용 |   스택   |   RStack  |
|-------|-------------|------|-----------|-----------|
|    0  |    BASEFEE  |    2 |        [] |        [] |
|    1  |    STOP     |    0 |       [7] |        [] |

출력: 0x
소비된 가스: `2`

## 보안 고려사항
기본 수수료 값은 민감하지 않으며 블록 헤더에서 공개적으로 접근 가능합니다. 이 옵코드와 관련하여 알려진 보안 문제는 없습니다.
